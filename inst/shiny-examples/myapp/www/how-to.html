Genoppi

Getting started

Genoppi is a user-friendly application integrating quantitative proteomics and genetic datasets that enables quality control steps to be performed on mass spectrometry data. Genoppi allows interactive overlaying of genetic datasets, such as GWAS, exome sequencing, the Exome Aggregation Consortium1 (ExAC) or other user-defined inputs with protein-protein interaction data. Genoppi can be fully deployed across Mac OS and Linux distributions for local installation for personal use. To do so:

  • Download and install RStudio and R version 3.3.x.
  • Begin RStudio and run the following commands to install necessary R packages

  • To start the local version of Genoppi, download the github repository, unzip and move the folder to a directory of your choice. Run the following commands in RStudio.

 

Please direct your questions and comments to aprilkim@broadinstitute.org, edytam@broadinstitute.org and/or lage.kasper@mgh.harvard.edu. 

Basics

Genoppi accepts i) experimental isobaric tag-based (iTRAQ or TMT) quantitative interaction proteomics data obtained from tandem mass spectrometry (MS/MS) analysis of immunoprecipitations (IPs) or ii) the MS/MS results from a whole proteome analysis. The input file must be a tab delimited plain text file. At minimum, the file must contain three columns specifying protein identifiers (either as gene symbols or as UniProt3 IDs [www.uniprot.org]), and two columns with protein enrichment values between studied conditions (as log2 fold change [FC]), one for each replicate.

  • Column 1: protein identifiers as either HUGO Gene Nomenclature Committee2 [HGNC] approved symbols, or UniProt3 [www.uniprot.org] accession numbers
  • Columns 2, 3: logFC values for two replicates observed in the IP experiment OR
  • Columns 2, 3, 4: results of 1-sample moderated t-test on two replicates: average logFC, P and false discovery rate (FDR) values
  • All fields must be filled with no missing values. Any rows with missing values will be disregarded with no error message.

Examples of correct column headers and accepted values:

1. HGNC symbol and logFC for two replicates

         

2. UniProt accession numbers and logFC for two replicates

         

3. HGNC symbol and results of statistical analyses

       

   

4. UniProt accession numbers and results of statistical analyses

           

 

If the protein enrichment values from two replicates are given, a moderated t-test from the limma15 package is applied to calculate P-values for the significance of a protein enrichment. Limma was originally introduced to robustly identify differentially expressed genes in microarray experiments and has since been used on a variety of data types, including proteomics results4,5. The empirical Bayes moderated t-test is used in Genoppi, because it is less sensitive to underestimated sample variances and performs best on small sample sizes compared to the classical t-test4.

If your data is in Excel format and you are a Mac user, please convert it to text file by selecting ‘File’ -> ‘Save As…’ and choose ‘Tab delimited Text (.txt)’ or ‘Comma Separated Values (.csv)’ as shown below. All other formats create a file that terminates each line with a carriage return character, which is incompatible with subsequent analysis in Genoppi.

Quality control

Upon data upload, interactive volcano and scatter plots will be generated as shown below. The scatter plot is only generated if the user input file includes rep1 and rep2 values.

The initial FDR threshold for color scale is 0.1. Results are shown as a volcano plot where logFC values of the bait versus control conditions are plotted on the x-axis and the negative log10 transformed P-values of enrichment are plotted on the y-axis. Enriched interaction partners with an FDR ≤ 0.1 are in green, and detected proteins with an FDR > 0.1 are in blue. Changing the FDR value threshold in the left panel will adapt colors in the interactive plot to reflect that change. The user is able to change the colors determined by the chosen FDR threshold using “Color selection for markers.”

Hovering over each dot, which represents a protein in the data, will show the corresponding HGNC gene symbol. The table to the right of the volcano plot shows the total count of enriched interactors in regions specified by the threshold controllers on the left panel.

Color by user value

The “user value” color scheme allows the user to upload genes and their associated values to be mapped on a color scale. The user can upload external datasets such as Project Achilles8,9,10,11, or RNA-Seq expression values associated with genes as a text file. The file must be in tabular format as delimited text with ‘gene’ and ‘score’ as column headers.

A discrete color scale assigns different colors to each score reported in the user file and a continuous color scale creates a gradient between the reported minimum and maximum scores. The color theme can then be changed using ‘color selection for markers.’ The example plot shown below is plotted using the YlGnBu (YellowGreenBlue) colormap.

Overlay InWeb

The user can further QC proteomics data by identifying known human protein interaction partners from InWeb_InBioMap7 (www.intomics.com/inbio/map.html#downloads), which contains known protein-protein interactions (PPI) compiled from >40,000 articles. Overlaying this information enables users to easily distinguish new interactions from those already reported in the literature. InWeb_InBioMap interactors of the bait protein are highlighted in yellow in the figure below. 

Overlay with genetics

Genoppi allows overlaying experimental proteomics data with

  • SNPs from genetic studies,
  • user-uploaded genes (e.g. set of interesting genes from Genotype Tissue Expression12 (GTEx, http://www.gtexportal.org/home/) to highlight corresponding proteins and
  • proteins encoded by genes that are intolerant to loss-of-function mutations as determined by ExAC data analysis.  

SNP-to-gene mapping

Genoppi hosts a pre-calculated pairwise measure of linkage disequilibrium (LD), which is used to perform SNP-to-gene mapping using the 1000 Genomes Project13 (http://www.internationalgenome.org) imputed single-nucleotide polymorphism (SNP) results. The user may provide a list of SNPs, found in the 1000 Genomes Project, that are significantly associated with a trait of interest to the experiment as a plain text file in the format shown below, with no column header.

The user may change the color theme for SNP markers by using the drop-down menu titled “SNPs”.

Genes of interest

This feature allows highlighting proteins corresponding to the user’s uploaded genes of interest (e.g. a set of genes from GTEx project, a set of genes from permutation-based P-value calculations). The user must upload the gene list with HGNC symbols as a plain text file. The column header must contain the name describing the group of genes as shown below. There is no limit to the number of columns.

A detailed count of overlaps and hypergeometric P-value is shown to the right of the volcano plot. The user may change the color scheme by using the drop-down menu titled ‘Genes of interest’.

The user may upload any combination of input files for SNP-to-gene mapping and genes of interest, and HGNC symbol for InWeb_InBioMap interaction partners. The resulting plot will highlight all identified proteins from user-input data. Overlaying multiple datasets could result in a densely overloaded plot, in which case the user can choose to remove labels or temporary hide markers by clicking on the legend to be hidden.

Intolerance to LoF mutations determined by ExAC data

If the user would like to identify proteins encoded by genes that are likely intolerant to loss-of-function variation using ExAC data, the user can select the “ExAC” color scheme on the left panel. Genes likely to be extremely intolerant to loss-of-function (LoF) variation are computed using ExAC pLI scores. Genes marked as pLI ≥ 0.9 are defined as extremely intolerant to LoF variations whereby genes marked as pLI ≤ 0.1 are defined as tolerant to LoF variations.

Significance of overlaps

When genetics datasets are integrated with proteomics data, the overlaps are statistically tested and the results are made available to users. Statistical significance can help determine, for example, the overlap of interactors and genes genetically linked to a particular disease. Hypergeometric distribution14 is used to perform gene set enrichment analyses between experimental proteomics data and 1) uploaded gene list(s) and/or 2) known protein-protein interactions from InWeb_InBioMap. The input file format used in the ‘Integrated Plots’ for genes of interest and InWeb_InBioMap input are identical to those used in the ‘Venn Diagrams’ tab.

Using genes of interest as an example, the user can adjust FDR, P-value and logFC sliders to select the values for significance threshold as shown in the figure below.

Upon threshold values selection, the overlap between experimental data and the InWeb database is identified to determine the population size. Successes in a population are determined as the proteins that passed the user-defined significance threshold. Next, the list of known interacting proteins of the user-defined bait is identified from the InWeb database and overlapped with the experimental data to determine the sample size. Lastly, proteins that are successes in the population and identified as InWeb interactors are determined as successes in the sample. Using this distribution, a hypergeometric test is run to calculate the statistical significance and the Venn diagram is generated to visualize the overlap between successes in population and sample. The user can adjust the analysis thresholds and have the changes (hypergeometric P-value and counts) reflected in the Venn diagram. The counts in each field of the Venn diagram are based on unique gene names.

1. P-value: hypergeometric P-value

2. population (N): subset of gene symbols from proteomics data found in HUGO Gene Nomenclature Committee (guidelines for Human Gene Nomenclature) (total number of genes)

3. number of successes in population (n): category A represents proteomics data present in HGNC database that passes the user-defined significance threshold (all significant bait interactors)

4. sample (S): category B represents user-uploaded genes found in the HGNC database OR InWeb_InBioMap interactors found in proteomics data  

If the user uploads multiple gene lists, denoted by different columns in one text file, Venn diagram representations and hypergeometric P-values for each list of genes will be generated and calculated, respectively. The user is able to browse through the multiple gene lists by clicking on the down arrow and selecting the category name, which corresponds to the column header uploaded by the user.

Protein families and subcellular localizations

Genoppi provides an option to overlay protein families or subcellular localizations to explore the diversity of protein functions in the proteomics results. An example of how protein families are displayed on a volcano plot is shown below..

To identify which protein families the identified proteins belong to, experimental pull-down data represented on a volcano and scatter plots can be overlaid with information on protein families of enriched genes based on user-defined significance (2).

By default, the ‘Turn on/off marker sizing’ option is set on ‘on’ and the user can change the markers size using the ‘Marker size’ slider, as shown below.

When “marker sizing” is turned on, it is a direct representation of how many proteins belong to the same protein family as indicated by different colors. Hovering over the marker will display protein identifiers using HGNC symbols, the protein family identified, and the number of other proteins belonging to the same protein family in the user-defined region.

Turning off “marker sizing” will display uniform-sized markers as shown in the figure below.

Protein families as scatter plot

This scatter plot is only applicable if the uploaded proteomics data contains two replicates, rep1 and rep2. Visualizing protein families in a scatter plot is especially useful when identifying general outliers and background contaminants, such as proteins that interact with affinity reagents or epitope tags. These outliers and contaminants often have poor correlation between two replicates and are plotted outside of the diagonal as shown below.

One method to troubleshoot and identify background contaminants from bona fide interactors is to identify which protein families the outliers (red box region above) belong to. Users can define the limits of rep1 and rep2 values using the appropriate sliders and use the preview area on the right to select a region of interest, as shown in the figure below.

Protein families removal

The “Protein families” field (1) in the Quality Control panel is a cleanup function designed to QC proteomics data by removing protein family members considered as contaminants or ‘sticky’ proteins. This feature is only available if the input data contains either HGNC approved symbols or UniProt accession numbers with logFC values for two replicates observed in the IP experiment.

The drop-down menu shows all protein families present in the HGNC database and more than one protein family can be selected. Any proteins in the dataset matching the selected protein family will be highlighted in the plot and appear in the legend. The matching protein families coloring can be changed using the ‘protein families color’ drop-down menu.

If there are any matches, the “Remove selected PF from input” button will appear, through which the user may download an updated input data where the proteins belonging to the selected protein families are removed.

Multiple files comparison (MFC)

Users can upload up to three proteomics experiment results to perform comparative analyses. The QC panel is similar to that of the single file upload and users can visualize up to three results as volcano and scatter plots, search for a protein of interest using its corresponding gene name, and search for and/or remove groups of genes belonging to specific protein families. Multiple files comparison features require a minimum of two input files. Users can change the marker colors by selecting the drop-down option titled ‘Color selection for markers.’ Genoppi interface with three loaded sample files is shown in the figure below.

Protein comparison

FDR, P-value and logFC controllers are generated for each uploaded proteomics dataset in the ‘Protein Comparison’ tab as shown below.

After setting the significance thresholds, clicking on the ‘Generate plots’ button will generate comparative volcano plots of all user-uploaded data. The comparison is performed using HGNC symbols as protein identifiers and the color scheme is as explained below.

f1: marked in red are proteins unique to file1

f2: marked in yellow are proteins unique to file2

f3: marked in blue are proteins unique to file3

f12: marked in orange are proteins identified in both file1 and file2

f13: marked in purple are proteins identified in both file1 and file3

f23: marked in green are proteins identified in both file2 and file3

f123: marked in gray are proteins identified in file1, file2 and file3

Volcano plots show the uploaded data as background markers in white/gray and colored markers belonging to different protein groups as explained above (i.e. unique to file1, in both file2 and file3 etc.)

To easily identify what proteins and how many of them are unique to each group, a Venn diagram represents protein distributions across files and the table to the right lists the proteins using HGNC symbols unique to each file.

MFC integrative analysis

The integrative analysis using genetics data and identifying protein families is performed similarly to the single file analysis. Please refer to the integrative analysis on the single file tab for a detailed description.

Downloads

Users can download all generated data as plain text files and interactive plots as HTML reports under the ‘Download’ tab. Download buttons are only active once the relevant data and/or plots have been generated.
